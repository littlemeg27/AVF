            Week One Research Topic

            Question One

            So today we have the ability to login to different sites like Pintrest, Groupon, Spotify with these sites and applications

            that we use almost everyday and with social networking sites like Facebook or twitter you can “login” so that you are using

            one universal username and password and the same with different payment options you would be doing the same thing.

            Question Two

            So according to Programmableweb.com there are 258 JSON-P API’s to the over a thousand JSON API’s and the reason we use or

            want to use JSON-P is that we can use the URL request to grab the data we need and then manipulate it the way we need

            right then and there.

            Question Three

            So from what I have seen JSON-P is very useful and used a lot in the geo-location aspect of the native device features to

            work with Google maps or other resources.

            Question Four

            Well last night I was working on my twitter API and I was trying to think about what I was going to pull in from twitter

            and my mom just so happened to put on a video by the Kid President so I figured I might just pull in some YouTube videos

            from him. As for design I like to keep it simple.

            Question Five

            So for the lessons learned regarding workflow. Once you think you have finished writing the code you are working on before

            running it always run it though a validator, if everything is working right then run the code so that you can test it even

            if it is HTML or CSS to see how it works on the visual aspect if you think it works then play around with it if its on an

            phone or tablet and think about how a user that isn’t experienced would think about using this app.

            Then change when and where it is needed.

            Week 2 Research Topic

            Question One

            So the first part of the ways you could change code to be more accessible would be first put it though a beautifier like

            http://tools.arantius.com/tabifier or http://www.digitalcoding.com/tools/html-beautifier.html and also make sure it is tabbed

            out nicely so that its easy to read. Second use semantic tags like header, section and footer so that your not using a billion

            div tags then comment out as needed to keep track of where everything is. The second part is the lay out of the app so that is

            follows the rules of usability like having back buttons, clearly marked out pages and buttons and also an escape hatch to get

            beck to the beginning of the app.

            Question Two

            So when i searched for support on a Mobile Screen Reader I didn’t really find much other then self made videos which kind of

            tells me that most of the instructions are probably online in writing which doesn’t make sense or you would have to spend

            forever on the phone. If I was to fix this I would make sure that it was a built in application or easy to download

            application with very few steps to set up and use.

            Question Three

            So I went into the Accessibility tab on my iPad it lists all kinds of modes to use for Accessibility like VoiceOver and

            Guided Access. Guided Access is so that when you want someone or something (kitten) to stay inside a certain application

            and cant escape of it you turn it on and have a few options when it comes down to motion or touch. The VoiceOver I didn’t

            get much of a chance to play with but I believe it is just like it says it helps with blind or near blind people to be

            able to use the ipad.

            The other device I saw was the Blackberry with the old OS, well old as of about two or three months ago. From what I saw

            on the videos for the blackberry Accessibility it took using a computer to setup the whole thing, but the good part of it

            was that it had keyboard shortcuts. But could read emails, calendars, Internet sites, and contacts.

            Question Four

            So for the iOS apps I found iBlink Radio, this application will read the newspaper and other publications. Also it will

            play music for the visually impaired.

            I was unable to really find anything in the Google Play Store, but from some research I was able to find Text to Speech

            Toy, which helps if you need to talk to someone who is deaf.
